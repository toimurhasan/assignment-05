Question-1: What are the different ways to select an element in the DOM?

There are 5 primary ways to select an element in the DOM. These are:

  1. document.getElementById('id') = returns a single element
  2. document.getElementsByClassName('class') = returns a HTMLCollection
  3. document.getElementsByTagName('tag') = returns a HTMLCollection
  4. document.querySelector('selector') = returns the first matching element
  5. document.querySelectorAll('selector') = returns a NodeList

Question-3: What is event delegation in the DOM?

Event delegation is a technique in JavaScript where a single event listener is attached to a parent element instead of attaching event listeners to multiple child elements. When an event occurs on a child element, the event bubbles up the DOM tree, and the parent element's event listener handles the event based on the target element.

In other words, instead of adding separate event listeners to many child elements (like buttons or list items), you add one event listener to their common parent. Then, when you click on any child, the event "bubbles up" to the parent, and the parent checks which child was clicked.

Question-4: What is event bubbling in the DOM?

Event Bubbling is a concept in the DOM where an event starts from the target element and then bubbles up (or propagates) to its parent elements, all the way up to the root element.

This means that when an event is triggered on a specific element, it first executes the event handler for that element, then propagates up to the parent element, and continues up the hierarchy until it reaches the root of the document.

This behavior allows parent elements to listen for events triggered on child elements, enabling techniques like event delegation. Event bubbling can be stopped using methods like event.stopPropagation(), preventing the event from propagating to ancestor elements.
